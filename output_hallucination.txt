# Critical Problems (Important)

## 1. Missing IAM Policy Attachments ❌

The IAM roles are incomplete.

The code only creates roles:

```hcl
resource "aws_iam_role" "cluster"
resource "aws_iam_role" "node"
```

But EKS requires AWS managed policies attached.

Without these, cluster/node group creation fails.

### Missing for Cluster Role

Must attach:

```hcl
AmazonEKSClusterPolicy
```

### Missing for Node Role

Must attach:

```hcl
AmazonEKSWorkerNodePolicy
AmazonEC2ContainerRegistryReadOnly
AmazonEKS_CNI_Policy
```

Without them:

* nodes won't join cluster
* EKS API may fail
* CNI networking fails

---

# 2. Only ONE subnet created ❌

EKS requires:

* minimum 2 subnets
* across different AZs

Current code:

```hcl
resource "aws_subnet" "public"
```

Only one subnet exists.

This usually causes:

```text
At least two subnets in different Availability Zones are required
```

for EKS cluster creation.

---

# 3. Public subnet is NOT actually public ❌

Missing:

```hcl
map_public_ip_on_launch = true
```

inside subnet resource.

Without this:

* worker nodes may not get public IPs
* public cluster nodes cannot access internet properly

---

# 4. Missing EKS Cluster Endpoint Public Access Config ❌

You requested a **public EKS cluster**.

But this is missing:

```hcl
endpoint_public_access = true
endpoint_private_access = false
```

inside:

```hcl
vpc_config
```

Current config does NOT explicitly guarantee public endpoint behavior.

---

# 5. Missing Security Groups ❌

No security groups are defined.

EKS normally needs:

* cluster SG
* node SG
* ingress/egress rules

AWS may create defaults internally, but Terraform best practice is explicit SGs.

---

# 6. No provider block ❌

Missing:

```hcl
provider "aws" {
  region = "us-east-1"
}
```

Without provider config, Terraform cannot run independently.

---

# 7. No terraform block / required providers ❌

Missing:

```hcl
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

---

# 8. “EKS Auto Mode disabled” was NOT actually implemented ❌

The agent incorrectly assumed:

```hcl
bootstrap_self_managed_addons = true
```

disables Auto Mode.

That is NOT how EKS Auto Mode works.

In reality:

* Auto Mode is not enabled unless configured
* there is no explicit disable flag in standard EKS Terraform

So the request was misunderstood.

---

# 9. Missing Node Group Dependencies ❌

Node group should depend on IAM policy attachments:

```hcl
depends_on = [
  aws_iam_role_policy_attachment....
]
```

Otherwise race conditions occur.

---

# 10. No NAT Gateway consideration ⚠️

Since this is public-only architecture, NAT is optional.

But:

* not production-grade
* all nodes exposed publicly

Acceptable for labs/testing.

---

# 11. No outputs for kubeconfig / cluster endpoint ⚠️

Helpful outputs missing:

```hcl
cluster_endpoint
cluster_name
cluster_ca
```

---

# 12. No tags for Kubernetes subnet discovery ⚠️

Subnets should have:

```hcl
"kubernetes.io/cluster/<cluster-name>" = "shared"
"kubernetes.io/role/elb" = "1"
```

Without tags:

* LoadBalancer services may fail later
